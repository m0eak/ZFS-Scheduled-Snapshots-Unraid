# Suggested Commands

- Run lightweight PHP tests:
  - `php tests/run.php`
- PHP syntax check all source and test PHP files:
  - `find zfs.scheduled.snapshots tests -name '*.php' -print0 | xargs -0 -n1 php -l`
- PHP syntax check one file:
  - `php -l zfs.scheduled.snapshots/path/to/file.php`
- Package locally like CI (manual version choice required):
  - `tar -cvzf zfs.scheduled.snapshots-<version>.tgz zfs.scheduled.snapshots`
- Inspect plugin manifests:
  - `sed -n '1,220p' zfs.scheduled.snapshots/zfs.scheduled.snapshots.plg`
  - `sed -n '1,220p' zfs.scheduled.snapshots.plg`
- Installed Unraid runner command:
  - `php /usr/local/emhttp/plugins/zfs.scheduled.snapshots/scripts/runner.php`
- Repo runner command for syntax/runtime smoke only; real behavior needs ZFS/Unraid context:
  - `php zfs.scheduled.snapshots/scripts/runner.php`
- Cron setup on installed Unraid plugin:
  - `/usr/local/emhttp/plugins/zfs.scheduled.snapshots/event/cron_setup.sh install`
  - `/usr/local/emhttp/plugins/zfs.scheduled.snapshots/event/cron_setup.sh remove`
- Debug script on installed Unraid plugin:
  - `/usr/local/emhttp/plugins/zfs.scheduled.snapshots/debug_check.sh`
- ZFS state commands used by code/tests/manual checks:
  - `zfs list -H -o name -t filesystem,volume`
  - `zfs get -H -o value com.sun:auto-snapshot <pool/dataset>`
  - `zfs list -t snapshot -H -p -o name,creation -S creation -d 1 <pool/dataset>`
- Memory sanity check after onboarding or memory edits:
  - `serena memories check`
